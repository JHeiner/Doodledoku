Click the little pencil button (near omnibox) to toggle doodling on/off.
  • go ahead: you can try it out now
  • green box means doodling is on, black means it is off
  • toggle doodling off to mouse things on the page
  • the box number indicates how many pages are being doodled on
  • if there are zero doodled pages the box is hidden

When doodling is on:
Drag the mouse to doodle.
  • here "drag" means "move mouse while holding down mouse button"
  • click (without drag) in empty area to make a dot
Click on doodles to erase them.
  • erasing is permanent (undo not available)
  • when doodles cross, clicking intersection erases all of them
Hold "control" key to erase more doodles.
  • doodles under mouse and any made since them are hilighted
  • ctrl+click erases all hilighted doodles
  • or ctrl+drag to sweep area & erase contained doodles
Keyboard focus follows the mouse.
  • doodles don't react to typing (just ctrl, as described above)
  • keyboard events go to the focusable elements on the page

Regardless of whether doodling is on/off:
Reloading or navigating away from a page erases all its doodles.
  • doodling is toggled off for that page
  • erasing is permanent (undo not available)

I enjoy playing Sudoku online, but it can get a little annoying because each site has a slightly different way to interact. On many (but not all) sites you can click and then type a number to solve a cell. But there are almost as many ways to mark candidates as there are sites with puzzles. And some sites don't even support marks! I created this extension to smooth over these differences so I don't have to worry as much about which site I'm on and I can instead just work on the puzzles.

I know if I print out the page I can make all those site differences completely disappear, and all I need to know is how to use a pencil. I'm sure you've guessed by now that this is the inspiration behind this extension. The pencil-and-paper interaction is fairly easy to implement, and also easy to understand and use. And it should not interfere much at all with most sites.

There are other extensions that let you doodle. Some even let you share your doodles with the world. And there are fully featured drawing programs that run in the browser. Those are fantastic extensions/apps, but they don't feel quite right for this task. They make the doodles the important thing when instead it should be the puzzle solving.

I should also point out that there are apps available (not necessarily browser based) specifically for solving puzzles. They are also fantastic, but I find much more satisfaction from doing all the work myself. So, in keeping with the inspiration, this extension helps to solve puzzles as much as pencil-and-paper does.

The simple way to think of the doodle button is as a toggle, but (of course) the reality is slightly more complicated. Some javascript code must be injected into a page to allow doodling, but it would be a waste to do that work for all the pages that never get doodled upon.
